01 — Compilation Pipeline

From text to tensors in one pass

A .hymeko source file compiles through six deterministic stages into content-addressable sparse tensors — no interpretation, no runtime overhead.

STAGE 01
SIMD Lexer
AVX2 / SSE2 / Scalar tiered tokenization. Zero-copy tokens borrowed from the input buffer.
STAGE 02
LALR Parser
LALRPOP-generated grammar produces a typed AST with signed references, annotations, and imports.
STAGE 03
Resolution
String interning, scope resolution via B-tree index, and recursive multi-file import loading.
STAGE 04
IR Lowering
Flat arena-based IR with linked-list children, jump tables, and O(1) type-specific lookup.
STAGE 05
Merkle Hash
Blake3 content-addressable identity per declaration. O(1) structural change detection.
STAGE 06
Tensor Extraction
Star expansion O(|E|·d) and clique expansion O(|E|·d²) into sparse COO / CSR formats.
02 — The Language

Describe systems, query patterns

The same LALR grammar that describes hypergraphs can describe patterns over them. A query is a partial description with wildcards.

robot_4wh.hymeko
// 4-wheel diff-drive robot description diff_robot use kinematics.elements { base_link: link { mass 25.0; link_geometry: box { dimension [0.7, 0.5, 0.2]; } } @joint_fr: conti_joint { + base_link, [[0.25, 0.25, 0.05], [-90.0, 0.0, 0.0]] - wheel_fr, - AXIS_Z } }

Signed incidence semantics

Nodes represent entities — links, sensors, layers. Edges (prefixed with @) represent multi-body relationships. The +, -, ~ operators encode directionality directly on the incidence connections.

In a robotic joint, + marks the parent link (source), - marks the child link (target) with an origin transform encoded as weights, and the axis is a separate - reference to an axis definition.

This 4-ary hyperedge captures what URDF flattens into XML attributes — the topology is the data.

find_joints.hymeko — query
// Find all continuous joints // connecting any two links query { @_ : conti_joint { + _ : link, - _ : link } }

05 — Application Domains

One framework, many fields

The same compilation pipeline serves robotics, machine learning, cognitive systems, virtual reality, knowledge representation, and runtime event detection.

🤖
Robotics
Kinematic chains as multi-body hyperedges. Joints connect parent links, child links, axes, and frames in a single 4-ary incidence. Transforms to URDF, SDF, ROS2.
🧬
Neural Architecture
Network layers as nodes, connections as hyperedges. Structural entropy feedback regularizes training. HyperKAN learns on compiled incidence topology with NURBS activations.
🧠
Cognitive Systems
CogInfoCom framework integration. Cognitive entities, capabilities, and interactions modeled as labeled hypergraphs with ontological hierarchy.
🌐
Virtual Reality
Scene graphs and multi-body simulations. Kinematic constraints as hyperedges connecting rigid bodies, frames, and interaction points. Transforms to glTF.
🗂️
Knowledge Graphs
Many-to-many relationships expressed natively as hyperedges. Tensor representations enable spectral clustering, centrality, and community detection.
⚡
Event Detection
Directed hyperedges as event templates with precondition tails and consequence heads. Observer-based runtime activation with structural change tracking.
06 — Research Foundation

Formally grounded

HyMeKo is the implementation of a doctoral dissertation with four proven theses.

THESIS 1
Cognitive aspects of intelligent robots can be described using hypergraphs. The LALR-based language is provably more concise than XML, JSON, and YAML.
THESIS 2
Labeled hypergraphs can be transformed into unique tensor representations via star and clique expansion. The transformation is universal and polynomial-time.
THESIS 3
Hypergraph descriptions can be transformed into domain-specific outputs via graph-filtering rules — unifying robotics, neural networks, VR, and cognitive systems.
THESIS 4
Structural entropy from hypergraph tensor representations can regularize neural network training, improving convergence stability and accuracy.
